Dev Notes — 2026-05-08
Unifying G-vector conventions between zero-field and Hofstadter codes
=====================================================================

Goal
----
The Hofstadter code (hamiltonian.py) assumes a specific q-vector ordering
baked into its chain matrices:

    q1 = ktheta * [0, -1]
    q2 = ktheta * [sqrt(3)/2, 1/2]
    q3 = ktheta * [-sqrt(3)/2, 1/2]

The zero-field code (zerofield.py) and the original MATLAB code use
G-vectors derived from the moire geometry, which come out rotated by 60
degrees and with opposite sign relative to the Hofstadter convention.
We want both codes to use the same convention so coupling matrices can
be shared.

Step 1: Deriving the effect of negating all G-vectors
-----------------------------------------------------
Starting from the moire potential expansion (Eq. 13 of arXiv:1406.0668):

    V(r) = sum_n [ T_n^dag e^{i G_n . r} + T_n e^{-i G_n . r} ]

If we replace G_n -> -G_n, the two exponentials swap roles, and the
coupling matrices transform as:

    T_n -> T_n^dag  (Hermitian conjugate)

This flips the sign of the psi phase (e^{i*psi} -> e^{-i*psi}) and
transposes the sublattice structure of each T matrix.

Step 2: Testing in MATLAB (generate_interlayer_hoppings_change_qs.m)
--------------------------------------------------------------------
Initial attempt: manually flipped psi (psi = -1*(-0.29)) AND took
conj(transpose(T)). This double-counted the psi flip — the conjugate
transpose already handles it. Bands were close but not identical.

Fix: keep psi = -0.29 as originally defined, and let conj(transpose)
do all the work. This gives T_new = T_old^dag with correct phases.

A second issue: the Q-vectors used to build the plane-wave basis must
NOT be negated, because they enter the Dirac Hamiltonian as (k - Q).
Negating Q changes H_Dirac and breaks eigenvalue preservation. Solution:
build Q vectors from the ORIGINAL q's before negation. The negated q's
only affect the hopping Kronecker deltas (which q-vector differences
connect which Q pairs).

Step 3: Cyclic permutation to match Hofstadter ordering
-------------------------------------------------------
After negation, the q-vectors are:

    -q1 ~ [sqrt(3)/2, 1/2]   (was [-sqrt(3)/2, -1/2])
    -q2 ~ [0, -1]            (was [0, 1])
    -q3 ~ [-sqrt(3)/2, 1/2]  (was [sqrt(3)/2, -1/2])
    (approximate directions; actual values from moire geometry)

To match the Hofstadter target ordering [0,-1], [sqrt(3)/2,1/2],
[-sqrt(3)/2,1/2], we apply the cyclic permutation:

    q1_new = q2_negated    (the [0,-1] one)
    q2_new = q3_negated    (the [sqrt(3)/2, 1/2] one)
    q3_new = q1_negated    (the [-sqrt(3)/2, 1/2] one)

The T matrices undergo the same cyclic permutation:

    T1_new = T2_negated,  T2_new = T3_negated,  T3_new = T1_negated

where T_negated means the conjugate-transposed version from Step 2.

This was implemented in generate_interlayer_hoppings_change_qs.m with
a block that saves originals, negates, permutes, and restores originals
for the k-path construction:

    % Save originals for k-path restoration
    q1_orig = q1; q2_orig = q2; q3_orig = q3; vb_orig = vb;

    % Negate G vectors and take Hermitian conjugate of T matrices
    q1 = -q1; q2 = -q2; q3 = -q3; vb = -vb;
    T1_K = conj(transpose(T1_K));  ...etc...

    % Cyclic permute to Hofstadter ordering
    tmp = q1; q1 = q2; q2 = q3; q3 = tmp;
    vb = vb([2,3,1], :);
    tmp = T1_K; T1_K = T2_K; T2_K = T3_K; T3_K = tmp;
    ...same for Kp...

    % (After hopping construction, restore originals for k-path)
    q1 = q1_orig; q2 = q2_orig; q3 = q3_orig; vb = vb_orig;

This was validated in MATLAB — bands match the original code exactly.

Step 4: Applying the same transformation to hamiltonian.py
----------------------------------------------------------
The Hofstadter code's chain matrices (_build_chain_matrices_K/Kp) are
already built for the target q-vector convention. The chain structure
encodes guiding-center hopping:

    chain1 (diagonal)      <-> q1 has q_x = 0
    chain2 (subdiagonal)   <-> q2 has q_x > 0
    chain3 (superdiagonal) <-> q3 has q_x < 0

So chain matrices did NOT need modification.

What changed in hamiltonian.py:

  get_interbilayerterms_K (lines 81-98):
    OLD q vectors: [sqrt(3)/2,-1/2], [0,1], [-sqrt(3)/2,-1/2]
    NEW q vectors: [0,-1], [sqrt(3)/2,1/2], [-sqrt(3)/2,1/2]

    OLD T matrices (K valley):
      t1 = v1*exp(+i*psi) * [[1, w^-1], [1, w^-1]]
      t2 = v1*exp(+i*psi) * [[1, w],    [w, w^-1]]
      t3 = v1*exp(+i*psi) * [[1, 1],    [w^-1, w^-1]]

    NEW T matrices (combined negate + cyclic permute = T_old_sigma(n)^dag):
      t1 = v1*exp(-i*psi) * [[1, w^-1], [w^-1, w]]
      t2 = v1*exp(-i*psi) * [[1, w],    [1, w]]
      t3 = v1*exp(-i*psi) * [[1, 1],    [w, w]]

  get_interbilayerterms_Kp (lines 101-118):
    Same NEW q vectors as K valley.

    OLD T matrices (Kp valley):
      t1 = v1*exp(-i*psi) * [[1, w],    [1, w]]
      t2 = v1*exp(-i*psi) * [[1, w^-1], [w^-1, w]]
      t3 = v1*exp(-i*psi) * [[1, 1],    [w, w]]

    NEW T matrices:
      t1 = v1*exp(+i*psi) * [[1, w],    [w, w^-1]]
      t2 = v1*exp(+i*psi) * [[1, w^-1], [1, w^-1]]
      t3 = v1*exp(+i*psi) * [[1, 1],    [w^-1, w^-1]]

  F_nm tables recompute automatically from the new q vectors via
  build_fnm_tables, so no changes needed there.
